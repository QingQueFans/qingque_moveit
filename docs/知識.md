📚 顶层代码 vs 方法内的代码
顶层代码 = 不在任何函数/类内部的代码
python
# ========== 顶层代码 ==========
import sys
import os

HAS_ML = False
CONFIG = {"max_iter": 100}

print("模块加载中...")

from ml_seed_predictor import MLSeedPredictor  # ← 这也是顶层代码

# ========== 不是顶层代码 ==========
class IKSolver:  # 类定义本身是顶层，但内部不是
    def __init__(self):  # 方法定义不是顶层
        print("初始化")  # 方法内的代码不是顶层
        
    def solve(self):  # 方法定义不是顶层
        x = 1 + 2  # 方法内的代码不是顶层

def helper():  # 函数定义是顶层
    print("helper")  # 函数内的代码不是顶层

# ========== 还是顶层代码 ==========
if __name__ == "__main__":  # 条件判断是顶层
    solver = IKSolver()  # 这里的代码会执行！
    result = solver.solve()
执行时机
顶层代码：模块被导入时立即执行

方法内的代码：只有方法被调用时才执行

你的情况
python
# ik_solver.py 中的顶层代码
from ml_seed_predictor import MLSeedPredictor  # ← 这是顶层代码，导入即执行！

class IKSolver:
    def __init__(self):
        self.ml_predictor = MLSeedPredictor()  # ← 这是方法内代码，创建实例时才执行
所以即使你不创建 IKSolver 实例，import 语句仍然会执行！
